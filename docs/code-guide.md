## 一句话概括

**CodeReviewBot = 一个24小时待命的代码审查员**  
当有人提交代码（Pull Request）时，它会自动检查并给出评论建议。

---

## 核心流程：从提交代码到收到评论

想象一下这个场景：

```
🏢 开发者提交代码  →  📬 GitHub 通知  →  🤖 CodeReviewBot 收到通知
        ↓                                        ↓
   获取代码差异                      AI 分析代码问题
        ↓                                        ↓
   整理评论格式                      📝 发表评论到 GitHub
```

### 详细流程图（文字版）

```
1️⃣ 开发者提交 PR
        ↓
2️⃣ GitHub 发送"有人提交了代码"的通知（WebHook）
        ↓
3️⃣ CodeReviewBot 接收通知
        ↓
4️⃣ 验证通知是真的（不是别人伪造的）
        ↓
5️⃣ 从 GitHub 获取代码改动内容（Diff）
        ↓
6️⃣ 把代码发送给 AI（GPT-4）分析
        ↓
7️⃣ AI 返回审查意见
        ↓
8️⃣ 把 AI 意见整理成漂亮的 Markdown 格式
        ↓
9️⃣ 把评论发到 GitHub 的 PR 页面上
        ↓
🔟 开发者收到通知，可以看到 AI 的建议
```

### 实际耗时

- **简单 PR（几十行代码）**：约 10-30 秒
- **复杂 PR（几百行代码）**：约 30-60 秒  
主要时间花在 AI 分析上

## 核心技术点：这些是做什么的？

### 1. WebHook - 接收通知的"门铃"

**通俗比喻**：  
WebHook 就像你家的**门铃**。当有人按门铃时，你会知道"有人来了"。

在代码世界里：
- 开发者提交 PR = 有人按门铃
- GitHub 会按响门铃（发送 HTTP 请求）
- 你的服务器打开门，看到"有新 PR 了！"

**技术细节**：
- 文件位置：`src/webhook/handler.js`
- 这是一个 `/webhook` 的网址，GitHub 会向这个地址发请求

### 2. 签名验证 - 确认"来的是真客人"

**通俗比喻**：  
你设定了一个**只有你知道的口令**。当门铃响时，你会先确认来人说对了口令，才开门。

这就是**安全保护**，防止坏人伪装成 GitHub 来发送假通知。

**技术细节**：
- 文件位置：`src/webhook/validator.js`
- 使用 HMAC-SHA256 算法验证
- 如果没有设置密钥，可以暂时跳过验证（但生产环境建议开启）

### 3. GitHub API - 用来"获取代码"和"发评论"

**通俗比喻**：  
GitHub API 就像是**遥控器**，让你可以远程操作 GitHub：
- 获取某个人提交了哪些代码改动
- 在 PR 下面发表留言

**技术细节**：
- 文件位置：`src/github/client.js`
- 使用 Octokit 库（GitHub 官方 SDK）
- 核心功能：
  - `getPullRequestDiff()` - 获取代码改动
  - `createReviewComment()` - 发表评论

### 4. LLM 调用 - 让 AI 看代码

**通俗比喻**：  
把代码递给一个**资深工程师**，让他帮忙 review。这个"资深工程师"就是 GPT-4。

**技术细节**：
- 文件位置：`src/analyzer/openai.js`
- 流程：
  1. 把代码格式化成 prompt
  2. 发送给 OpenAI API
  3. 解析 AI 返回的 JSON 结果

### 5. 评论格式化 - 把 AI 输出变漂亮

**通俗比喻**：  
AI 返回的是原始笔记，需要**排版成漂亮的文档**，加上 emoji、分级、颜色等。

**技术细节**：
- 文件位置：`src/comment/formatter.js`
- 功能：
  - 按严重程度分组（高/中/低）
  - 加上 emoji 图标
  - 生成 Markdown 格式

## 常见问题解答

### Q: 这个服务需要一直运行吗？
A: 是的，需要一台服务器 24 小时运行，接收 GitHub 的通知。

### Q: AI 会看我的代码吗？安全吗？
A: 代码会发送给 OpenAI（GPT-4）进行分析。建议使用公司账号或确保代码不涉及敏感信息。

### Q: 可以审查所有编程语言吗？
A: 可以，GPT-4 支持多种语言。但对于特别小众的语言，可能效果一般。

### Q: 评论是怎么判断重要程度的？
A: AI 会自己判断，标记为 "high"（高）、"medium"（中）、"low"（低）三个级别。

## 总结

CodeReviewBot 是一个**自动化代码审查工具**，工作原理简单来说：

1. **有人提交代码** → GitHub 通知
2. **收到通知** → 验证是真的
3. **获取代码** → 调用 GitHub API
4. **AI 审查** → 发送给 GPT-4
5. **返回评论** → 整理格式，发到 PR 页

整个过程**全自动**，开发者会像收到邮件一样收到 AI 的审查建议。
