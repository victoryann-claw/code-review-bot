# AI 代码审查 Agent - 需求文档

## 1. 项目概述

### 1.1 项目名称
**CodeReviewBot** - AI 驱动的代码审查助手

### 1.2 项目目标
开发一个自动化的 AI 代码审查工具，通过 GitHub Webhook 接收 PR 事件，调用 LLM 对代码进行多维度分析，并自动生成审查评论。

### 1.3 目标用户
- 个人开发者
- 小型开发团队
- 开源项目维护者

---

## 2. 功能需求

### 2.1 核心功能

#### F1: GitHub Webhook 接收
- 监听 GitHub PR 事件（opened, synchronize）
- 验证 Webhook 请求签名
- 解析 PR 元数据（作者、文件列表、diff）

#### F2: 代码获取
- 通过 GitHub API 获取 PR 中的文件变更
- 支持多种编程语言
- 限制单次审查文件数量（防超限）

#### F3: AI 审查分析
- 调用 LLM API 进行代码分析
- 多维度审查维度：
  - 代码规范（ESLint/Prettier 规则）
  - 潜在 bug
  - 性能优化建议
  - 安全漏洞检测
  - 代码重复
  - 最佳实践
- 生成结构化审查报告

#### F4: 自动评论
- 通过 GitHub API 添加 PR 评论
- 支持 Markdown 格式
- 支持代码片段高亮
- 可配置评论位置（PR 总体 / 单文件）

#### F5: 配置管理
- 支持黑名单（忽略某些仓库/用户）
- 支持关键词过滤
- 支持审查深度级别（简单/标准/深入）

### 2.2 非功能需求

#### 性能
- 单次 PR 审查响应时间 < 30秒
- 支持并发处理多个 PR

#### 可用性
- 7x24 运行
- 部署简单，一键启动

#### 可扩展性
- 支持多种 GitHub 托管平台（GitHub.com / GitHub Enterprise）
- 支持多种 LLM 提供商

---

## 3. 用户流程

```
1. 用户在 GitHub 仓库配置 Webhook
2. 开发者提交 PR
3. Webhook 触发 → Server 接收事件
4. Server 获取 PR diff
5. 调用 LLM 分析代码
6. 生成审查报告
7. 通过 GitHub API 添加评论
8. 用户收到通知
```

---

## 4. 验收标准

- [ ] 能接收 GitHub Webhook 事件
- [ ] 能正确获取 PR 文件变更
- [ ] 能调用 LLM 进行代码分析
- [ ] 能自动生成审查评论
- [ ] 支持主流编程语言（JS/TS/Python/Java/Go）
- [ ] 部署文档完整，可复现
- [ ] 单元测试覆盖率 > 60%

---

## 5. 风险与限制

- **API 配额**：LLM API 有调用限制，需考虑成本
- **敏感数据**：代码可能包含敏感信息，需确保安全
- **误判**：AI 审查非完美，需提供忽略机制
